# 오브젝트 풀링 기반 대화 기록 UI 시스템

## 기능 개요
대화 기록 UI에서 메시지 오브젝트를 오브젝트 풀링으로 재사용하여 생성/삭제 비용을 줄이고 성능을 최적화한 시스템

## 핵심 흐름
1. 진행했던 대사의 위치 정보(BlockID)를 기반으로 표시할 대사 범위 결정
2. ScriptData에서 해당 대사 데이터를 로드
3. 메시지 오브젝트를 생성하거나 풀에서 재사용하여 UI 구성
4. 스크롤 확장 시 기존 오브젝트를 재활용하여 추가 대사 출력

## 주요 코드
DialogueLog.cs → 현재 띄울 대사들의 위치를 마킹하고, 대사 데이터 로드 후 메시지 오브젝트 생성

DialogueLogChattingArea.cs → 전달받은 대사 데이터에 따라 메시지 오브젝트 구성
